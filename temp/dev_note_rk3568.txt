 ====
Implement SDIO hang detection and send uevent (no sysfs polling).

Target:
- Android BSP kernel Linux 4.x
- WiFi SDIO device on mmc0
- RTL8822ES vendor driver (rtw_sdio_raw_write exists)

Hang rule:
- if SDIO operation returns -ETIMEDOUT (-110),
  count consecutive timeouts within a 10-second window.
- if 3 consecutive timeouts happen within 10 seconds:
  set hang_flag=1 and send uevent once for this hang event.
- clear hang_flag and counters on first successful SDIO transaction.

Send uevent:
- use kobject_uevent_env() with KOBJ_CHANGE
- use an existing device kobject (prefer mmc host or the SDIO function device)
- include these env variables:
  FAULT_ID=SDIO_HANG
  DEVICE=mmc0
  ERRNO=-110
  COUNT=<consecutive_count>
  TS_MS=<ktime_get_boottime_ns()/1e6>

Debounce:
- do not spam: send only once per hang event (until hang clears then happens again)

Provide:
- data structures (counters, timestamps, hang_flag)
- exact code to build envp[] and call kobject_uevent_env()
- integration point in rtw_sdio_raw_write() (and/or common SDIO read/write wrapper)
- thread safety (spinlock or mutex)

Implement an Android 12 system_server component that listens to kernel uevents
and forwards HW fault events to a privileged app.

Requirements:
1) Open NETLINK_KOBJECT_UEVENT socket (uevent) and parse messages.
2) Filter only events with:
   FAULT_ID=SDIO_HANG and DEVICE=mmc0
3) On first receipt of a new hang event:
   - send protected broadcast:
     Action: com.kdnavien.intent.action.HW_FAULT
     Permission: com.kdnavien.permission.HW_FAULT
     Extras:
       fault_id="SDIO_HANG"
       device="mmc0"
       errno=int(ERRNO)
       count=int(COUNT)
       driver_ts_ms=long(TS_MS)
       timestamp_ms=System.currentTimeMillis()
4) Prevent broadcast spam:
   - keep last seen driver_ts_ms, ignore duplicates
5) Provide:
   - uevent socket receiver thread (HandlerThread or dedicated Thread)
   - uevent parser utility
   - broadcast sending code (sendBroadcastAsUser)
   - service wiring notes
